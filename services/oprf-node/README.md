# World ID OPRF Node

The World ID OPRF Node is the protocol's OPRF service implementation. The
workspace package for the service is `world-id-oprf-node`.

## Releases

`world-id-oprf-node` is released independently from the published Rust crates. Releases are
triggered by pushing a `world-id-oprf-node-vX.Y.Z` tag that points to a commit on `main`.

1. Bump `version` in `services/oprf-node/Cargo.toml` to the desired version and refresh the
   lockfile (`cargo update -p world-id-oprf-node`). The Docker build runs with `--locked`, so
   the lockfile must be up to date.
2. Commit the version bump to `main` via a PR.
3. After the PR is merged, tag the merge commit and push the tag:
   ```sh
   git tag world-id-oprf-node-vX.Y.Z <commit-sha>
   git push origin world-id-oprf-node-vX.Y.Z
   ```
4. The `Release OPRF Node` workflow will:
   - Verify the tag points to a commit on `main` (tags on side branches are rejected).
   - Build and push the Docker image to GHCR (`ghcr.io/worldcoin/world-id-protocol/world-id-oprf-node`).
   - Cut a GitHub release with release notes generated by git-cliff, scoped to commits under
     `services/oprf-node/` since the previous `world-id-oprf-node-v*` tag.
   - No staging deploy is triggered (that happens separately in the deploy repo).

Tags with a pre-release suffix (e.g. `world-id-oprf-node-v2.0.0-rc.1`) are published as
pre-releases on GitHub and do not receive the `latest` Docker tag.
